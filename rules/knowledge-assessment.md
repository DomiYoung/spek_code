# Knowledge Value Assessment

> 知识价值评估体系，由 RULES.md 引用
> 基于 [Second Brain](https://buildingasecondbrain.com) 方法论、[Anthropic AI Agent 最佳实践](https://anthropic.com)

## 强制行为

任务完成后，**自动评估**是否需要记录到 KI 知识库。

## 评估维度

满足 2+ 项即记录：

| 维度 | 说明 | 权重 |
|------|------|------|
| 🔍 隐蔽性 | 问题不明显，容易忽略 | +1 |
| ⏱️ 耗时性 | 调试/定位 > 15 分钟 | +1 |
| 🔄 复发性 | 同一问题出现 2+ 次 | +2 |
| 🌐 跨项目 | 可能在其他项目重现 | +1 |
| 🤯 反直觉 | 违背常规预期的行为 | +1 |
| 📚 教育性 | 对团队有学习价值 | +1 |
| 🔗 关联性 | 可形成知识网络 | +1 |
| 🛡️ 预防性 | 可预防未来问题 | +1 |

## 价值等级判定

```
总分 1-2 = ⭐     可选记录
总分 3-4 = ⭐⭐   建议记录
总分 5+  = ⭐⭐⭐ 强制记录
```

## PARA 分类法

- **Projects**: 项目特定临时知识
- **Areas**: 长期维护领域知识
- **Resources**: 可复用通用知识（pitfalls.md）
- **Archives**: 历史知识归档

## 自动触发条件

1. Bug 修复完成后
2. 跨文件/跨分支对比才能定位的问题
3. 需要 Web 搜索才解决的问题
4. 用户明确表示"踩坑"、"坑"、"记录"
