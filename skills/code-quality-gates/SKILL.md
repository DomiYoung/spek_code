# Code Quality Gates

## 操作规则

### 必须做 ✅
- **Read-Before-Write**: 修改前必须先读取现有代码
- **绝对路径**: 始终使用绝对路径
- **批量操作**: 尽量批量处理，减少单次操作
- **验证优先**: 执行前验证，完成后确认
- **发现优先**: 完整探索后再修改

### 禁止做 ❌
- 跳过 Read 直接 Write/Edit
- 使用相对路径
- 未经授权的自动提交
- 忽略框架模式
- 跳过验证步骤

## 完成标准

### 代码提交前检查
- [ ] `lint` 通过
- [ ] `typecheck` 通过
- [ ] 相关测试通过
- [ ] 无控制台调试语句
- [ ] 无硬编码凭据

### 功能验收标准
- [ ] 符合原始需求
- [ ] 边界情况处理
- [ ] 错误处理完善
- [ ] 文档/注释完整

## 任务状态管理
```
📋 pending    - 待执行
🔄 in_progress - 执行中 (单任务)
🚧 blocked    - 等待依赖
✅ completed  - 已完成
```

## 严重度分级
| 级别 | 处理时间 | 影响 |
|------|---------|------|
| CRITICAL | 立即 | 阻塞发布 |
| HIGH | 24h | 需修复 |
| MEDIUM | 7d | 计划修复 |
| LOW | 30d | 可选修复 |
