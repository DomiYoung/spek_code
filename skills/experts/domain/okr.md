## OKR 核心知识库

> 基于 John Doerr《Measure What Matters》、Andy Grove《High Output Management》、Christina Wodtke《Radical Focus》

### 1. OKR 核心概念

**定义**：
- **O (Objective)**：目标 - 定性、鼓舞人心、有挑战性
- **KR (Key Result)**：关键成果 - 定量、可衡量、验证目标达成

**核心原则**：
| 原则 | 说明 |
|------|------|
| **聚焦** | 每层级 3-5 个 O，每个 O 配 3-5 个 KR |
| **对齐** | 自上而下对齐战略，自下而上支撑目标 |
| **透明** | 全公司可见，促进协作 |
| **挑战** | 60-70% 完成率 = 设置得当（登月目标）|
| **解耦** | OKR 与薪酬/晋升分离 |

### 2. OKR 制定框架

**优秀 Objective 特征**（CIVIC 原则）：
```
C - Challenging（有挑战性）
I - Inspiring（鼓舞人心）
V - Valuable（有价值）
I - Important（重要紧急）
C - Clear（清晰易懂）
```

**优秀 Key Result 特征**（SMART-M）：
```
S - Specific（具体）
M - Measurable（可衡量）
A - Achievable（可实现，但有挑战）
R - Relevant（相关性）
T - Time-bound（有时限）
M - Metrics-based（基于指标）
```

### 3. OKR 类型

| 类型 | 完成率目标 | 适用场景 |
|------|-----------|---------|
| **承诺型 (Committed)** | 100% | 必须完成的基础目标 |
| **挑战型 (Aspirational)** | 60-70% | 登月目标，推动极限 |
| **学习型 (Learning)** | N/A | 探索新领域，验证假设 |

### 4. OKR 周期与节奏

```
年度 OKR（战略方向）
    ↓ 分解
季度 OKR（执行重点）
    ↓ 追踪
周度 Check-in（进度同步）
    ↓ 反馈
季度复盘（回顾评分）
```

**推荐节奏**：
| 活动 | 频率 | 时长 | 参与者 |
|------|------|------|--------|
| OKR 制定 | 季度初 | 2-3 天 | 全员 |
| Check-in | 每周 | 15-30 分钟 | 团队 |
| Mid-term Review | 季度中 | 1 小时 | 管理层 |
| 复盘评分 | 季度末 | 半天 | 全员 |

### 5. OKR 评分标准

**Google 评分法（0.0-1.0）**：
```
0.0-0.3 = 🔴 未达预期（需复盘原因）
0.4-0.6 = 🟡 部分完成（正常区间）
0.7-1.0 = 🟢 超额完成（可能目标过低）
```

**颜色编码**：
```
🔴 红色 (0-30%): 严重偏离，需要干预
🟡 黄色 (30-70%): 有风险，需要关注
🟢 绿色 (70-100%): 进展顺利
```

### 6. CFR 持续反馈

**CFR = Conversation + Feedback + Recognition**

| 组件 | 说明 | 频率 |
|------|------|------|
| **Conversation** | 1:1 对话，讨论进展与障碍 | 每周/双周 |
| **Feedback** | 双向反馈，及时调整 | 持续 |
| **Recognition** | 认可贡献，激励团队 | 随时 |

### 7. OKR 对齐模式

```
        ┌─────────────────┐
        │   公司 OKR      │  ← 战略方向（CEO/高管）
        │  3-5 个目标     │
        └────────┬────────┘
                 │ 分解
    ┌────────────┼────────────┐
    ▼            ▼            ▼
┌────────┐  ┌────────┐  ┌────────┐
│部门 OKR│  │部门 OKR│  │部门 OKR│  ← 支撑公司目标
└───┬────┘  └───┬────┘  └───┬────┘
    │           │           │
    ▼           ▼           ▼
┌────────┐  ┌────────┐  ┌────────┐
│团队 OKR│  │团队 OKR│  │团队 OKR│  ← 支撑部门目标
└───┬────┘  └───┬────┘  └───┬────┘
    │           │           │
    ▼           ▼           ▼
┌────────┐  ┌────────┐  ┌────────┐
│个人 OKR│  │个人 OKR│  │个人 OKR│  ← 支撑团队目标
└────────┘  └────────┘  └────────┘
```

**对齐方式**：
- **垂直对齐**：上级 KR → 下级 O
- **水平对齐**：跨团队协作的共享 KR

### 8. 常见 OKR 模板

**技术团队示例**：
```markdown
## Q1 2025 Engineering OKR

### O1: 提升平台稳定性，让用户无感知故障
- KR1: 系统可用性从 99.5% 提升至 99.9%
- KR2: P0 故障平均恢复时间从 30 分钟降至 10 分钟
- KR3: 关键路径自动化测试覆盖率达到 80%

### O2: 加速产品迭代，缩短用户价值交付周期
- KR1: 需求到上线平均周期从 14 天降至 7 天
- KR2: 代码评审平均响应时间从 4 小时降至 1 小时
- KR3: CI/CD 流水线成功率从 85% 提升至 95%

### O3: 建设技术影响力，打造行业标杆工程文化
- KR1: 完成 3 篇高质量技术博客发布
- KR2: 主导或参与 2 个开源项目贡献
- KR3: 团队成员技术分享覆盖率达到 100%
```

**产品团队示例**：
```markdown
## Q1 2025 Product OKR

### O1: 提升用户留存，让用户持续从产品获得价值
- KR1: 7 日留存率从 40% 提升至 55%
- KR2: 月活跃用户 MAU 增长 30%
- KR3: NPS 评分从 35 提升至 50

### O2: 优化核心体验，减少用户完成任务的摩擦
- KR1: 核心流程完成率从 60% 提升至 80%
- KR2: 用户平均任务完成时间减少 25%
- KR3: 用户投诉率下降 40%
```

**个人 OKR 示例**：
```markdown
## Q1 2025 个人 OKR

### O1: 成为团队可依赖的技术核心成员
- KR1: 独立负责并交付 2 个核心模块
- KR2: 代码评审参与率达到 90%，发现 10+ 个关键问题
- KR3: 完成 1 次全组技术分享，满意度 ≥ 4.5/5

### O2: 提升专业技能，扩展技术广度
- KR1: 完成 React 高级模式学习，输出 3 篇笔记
- KR2: 掌握 GraphQL，在项目中实践 1 个 API
- KR3: 阅读 2 本技术书籍并输出读书笔记
```

### 9. OKR 常见反模式

| 反模式 | 问题 | 改进建议 |
|--------|------|---------|
| **KPI 化** | KR 变成考核指标，团队畏惧挑战 | 明确 OKR 不与薪酬挂钩 |
| **过多目标** | 焦点分散，什么都做不好 | 每层级最多 3-5 个 O |
| **输出导向** | 只关注做了什么，忽略结果 | KR 必须是结果，非任务 |
| **不透明** | 只有管理层知道 OKR | 全员可见，公开评分 |
| **一成不变** | 季度中不调整，脱离实际 | 允许 Mid-term 调整 |
| **自上而下** | 完全由管理层制定 | 60% 自下而上提出 |

### 10. OKR vs KPI 对比

| 维度 | OKR | KPI |
|------|-----|-----|
| **目的** | 激励挑战，追求卓越 | 考核绩效，保障底线 |
| **达成率** | 60-70% 为健康 | 100% 为目标 |
| **透明度** | 全员可见 | 通常保密 |
| **薪酬关联** | 不挂钩 | 直接挂钩 |
| **调整频率** | 可以季中调整 | 通常固定 |
| **适用场景** | 创新、增长 | 运营、合规 |

**混合使用建议**：
```
KPI（守住底线）+ OKR（追求突破）= 双轨制
```

### 14. OKR 实施路线图

```
Phase 1: 试点（1-2 个季度）
├─ 选择 1-2 个团队试点
├─ 培训 OKR 理念和方法
├─ 简化流程，允许犯错
└─ 收集反馈，迭代改进

Phase 2: 推广（2-3 个季度）
├─ 扩展到更多团队
├─ 建立对齐机制
├─ 引入工具支持
└─ 培养 OKR 教练

Phase 3: 成熟（持续）
├─ 全员 OKR 覆盖
├─ 与战略规划深度整合
├─ 形成 OKR 文化
└─ 持续优化节奏和流程
```
