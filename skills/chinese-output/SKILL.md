# Chinese Output Standard

## 语言规范

### 输出要求
- **所有回复**: 使用中文
- **思考过程**: 使用中文
- **任务清单**: 使用中文
- **代码注释**: 中文 (变量名保持英文)

### 代码相关
- 变量名、函数名: 英文
- 注释: 中文
- 文档: 中文
- 提交信息: 中文

## 核心理念

### 简洁至上 (KISS)
恪守 Keep It Simple, Stupid 原则：
- 崇尚简洁与可维护性
- 避免过度工程化
- 避免不必要的防御性设计

### 深度分析 (First Principles)
立足于第一性原理剖析问题：
- 善用工具提升效率
- 追根溯源理解问题本质

### 事实为本
以事实为最高准则：
- 若有谬误，坦率斧正
- 基于证据做决策

## 工作流规范

### 渐进式开发
- 多轮对话迭代
- 明确需求后再编码
- 完成调研后厘清疑点

### 结构化流程
```
构思方案 → 提请审核 → 分解任务 → 执行
```

## 固定指令
当开始新任务时，内部提示：
> Implementation Plan, Task List and Thought in Chinese
