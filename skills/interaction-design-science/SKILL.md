# 交互设计科学

## 认知心理学基础

### 认知负荷理论 (Cognitive Load Theory)

| 类型 | 说明 | 设计策略 |
|------|------|---------|
| **内在负荷** | 任务固有复杂度 | 分步骤、渐进披露 |
| **外在负荷** | 不良设计导致 | 简化界面、去除干扰 |
| **相关负荷** | 学习与理解 | 提供反馈、建立心理模型 |

### 工作记忆限制

```
Miller's Law: 7 ± 2 项目
├── 导航菜单: 5-7 项
├── 表单字段: 分组不超过 7 个
├── 选项列表: 分类展示
└── 信息架构: 3 层以内
```

### 注意力与感知

| 概念 | 说明 | 应用 |
|------|------|------|
| 选择性注意 | 只能聚焦有限信息 | 视觉层级、焦点设计 |
| 变化盲视 | 忽略渐变改变 | 明确状态变化、动画提示 |
| 非注意盲视 | 专注时遗漏其他 | 关键信息显著位置 |
| 模式识别 | 快速识别熟悉模式 | 一致的设计语言 |

## 交互设计定律

### Fitts' Law (费茨定律)

```
T = a + b × log₂(D/W + 1)

T: 移动时间
D: 距离
W: 目标宽度

设计启示：
├── 重要按钮放大
├── 常用功能靠近
├── 利用屏幕边缘 (无限宽度)
└── 减少精确操作
```

### Hick's Law (希克定律)

```
T = b × log₂(n + 1)

T: 决策时间
n: 选项数量

设计启示：
├── 减少选项数量
├── 分类组织信息
├── 提供默认选项
└── 渐进式披露
```

### Jakob's Law (雅各布定律)

```
用户期望你的网站与他们已知的网站相似运作

设计启示：
├── 遵循平台惯例
├── 标准化交互模式
├── 符合心理模型
└── 创新需渐进引入
```

### Law of Proximity (接近法则)

```
空间上接近的元素被视为相关

设计启示：
├── 相关元素紧密排列
├── 使用留白分隔组
├── 表单字段分组
└── 层级关系可视化
```

### Law of Similarity (相似法则)

```
视觉上相似的元素被视为相关

设计启示：
├── 一致的按钮样式
├── 统一的图标风格
├── 颜色编码分类
└── 重复设计模式
```

### Tesler's Law (复杂度守恒)

```
每个应用程序都有固有复杂度，无法消除

设计启示：
├── 让系统承担复杂度
├── 智能默认值
├── 自动化常见操作
└── 复杂度转移而非消除
```

## 可用性原则

### Nielsen's 10 Heuristics

| # | 原则 | 说明 |
|---|------|------|
| 1 | **系统状态可见** | 及时反馈当前状态 |
| 2 | **系统与现实匹配** | 使用用户熟悉的语言 |
| 3 | **用户控制与自由** | 提供撤销和退出 |
| 4 | **一致性与标准** | 遵循平台惯例 |
| 5 | **错误预防** | 防止问题发生 |
| 6 | **识别胜于记忆** | 减少记忆负担 |
| 7 | **灵活性与效率** | 为专家提供快捷方式 |
| 8 | **美学与极简** | 去除无关信息 |
| 9 | **帮助用户识别错误** | 明确错误原因 |
| 10 | **帮助与文档** | 易于搜索的帮助 |

### Don Norman's 设计原则

```
可见性 (Visibility)
├── 操作控件可见
├── 状态明确显示
└── 功能一目了然

反馈 (Feedback)
├── 操作即时响应
├── 进度明确展示
└── 结果清晰告知

约束 (Constraints)
├── 限制错误操作
├── 引导正确路径
└── 减少决策负担

映射 (Mapping)
├── 控件与效果对应
├── 物理隐喻清晰
└── 空间关系直观

一致性 (Consistency)
├── 内部一致
├── 外部一致
└── 操作一致

可供性 (Affordance)
├── 外观暗示功能
├── 行为符合预期
└── 交互方式自然
```

## 交互模式库

### 导航模式

| 模式 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| 标签导航 | 3-5个主要功能 | 直观、持久 | 空间有限 |
| 汉堡菜单 | 次要功能 | 节省空间 | 可发现性低 |
| 底部导航 | 移动端主导航 | 易于触达 | 占用空间 |
| 面包屑 | 层级内容 | 位置感知 | 不适合深层 |
| 侧边栏 | 桌面端 | 功能丰富 | 移动端困难 |

### 输入模式

| 模式 | 适用场景 | 要点 |
|------|---------|------|
| 表单填写 | 数据收集 | 分步骤、即时验证 |
| 搜索 | 内容发现 | 自动建议、筛选 |
| 选择器 | 选项选择 | 适当数量、默认值 |
| 拖放 | 排序/组织 | 可视化反馈 |
| 语音输入 | 解放双手 | 错误容忍 |
| 手势 | 快捷操作 | 可发现性 |

### 反馈模式

| 类型 | 用途 | 示例 |
|------|------|------|
| 即时反馈 | 操作确认 | 按钮状态变化 |
| 进度反馈 | 等待状态 | 进度条、加载动画 |
| 确认反馈 | 重要操作 | 确认对话框 |
| 错误反馈 | 问题提示 | 行内错误信息 |
| 成功反馈 | 完成通知 | Toast 提示 |
| 空状态 | 无数据 | 引导用户下一步 |

## 用户研究方法

### 定性方法

| 方法 | 目的 | 产出 |
|------|------|------|
| 用户访谈 | 深入理解需求 | 洞察报告 |
| 情境调研 | 观察真实使用 | 行为模式 |
| 焦点小组 | 群体观点 | 主题归纳 |
| 卡片分类 | 信息架构 | 分类结构 |
| 可用性测试 | 发现问题 | 问题列表 |

### 定量方法

| 方法 | 目的 | 产出 |
|------|------|------|
| A/B 测试 | 方案对比 | 统计显著性 |
| 问卷调查 | 规模化收集 | 统计数据 |
| 数据分析 | 行为追踪 | 使用模式 |
| 眼动追踪 | 视觉注意力 | 热力图 |
| 基准测试 | 效率测量 | 任务指标 |

### 可用性指标

```
效率 (Efficiency)
├── 任务完成时间
├── 操作步骤数
└── 错误率

有效性 (Effectiveness)
├── 任务成功率
├── 错误恢复率
└── 目标达成度

满意度 (Satisfaction)
├── SUS 评分 (System Usability Scale)
├── NPS 净推荐值
└── 主观评价
```

| 层次 | 特点 | 设计关注点 |
|------|------|-----------|
| **本能层** | 第一印象 | 视觉吸引力、感官愉悦 |
| **行为层** | 使用体验 | 效率、可用性、功能性 |
| **反思层** | 意义价值 | 品牌认同、自我表达 |

```

反馈 (Feedback)

循环与模式 (Loops & Modes)
```

## 无障碍设计 (A11Y)

### WCAG 2.1 原则

```
POUR 原则:
├── Perceivable (可感知)
│   ├── 替代文本
│   ├── 时间性媒体替代
│   ├── 适应性呈现
│   └── 可区分内容
│
├── Operable (可操作)
│   ├── 键盘可访问
│   ├── 足够时间
│   ├── 癫痫安全
│   └── 可导航
│
├── Understandable (可理解)
│   ├── 可读性
│   ├── 可预测
│   └── 输入辅助
│
└── Robust (健壮性)
    ├── 兼容性
    └── 辅助技术支持
```

### 核心检查项

| 类别 | 要求 | 标准 |
|------|------|------|
| 颜色对比 | 文本可读 | 4.5:1 (正常), 3:1 (大文本) |
| 焦点指示 | 可见焦点 | 明确的焦点样式 |
| 替代文本 | 图片描述 | 有意义的 alt |
| 表单标签 | 关联标签 | label for |
| 键盘访问 | 完全可操作 | Tab 顺序合理 |
| 语义化 | 正确标记 | 使用语义 HTML |

## 设计系统构建

### 原子设计方法论

```
原子 (Atoms)
├── 颜色
├── 字体
├── 图标
├── 间距
└── 按钮

分子 (Molecules)
├── 搜索框 (输入 + 按钮)
├── 导航项 (图标 + 文字)
└── 表单项 (标签 + 输入 + 错误)

有机体 (Organisms)
├── 头部导航
├── 产品卡片
└── 评论区

模板 (Templates)
├── 页面布局
└── 内容区域

页面 (Pages)
├── 具体内容
└── 真实数据
```
